# ドキュメント（プロジェクトルート整理）

このフォルダに、仕様・ヒアリング・参考資料・サンプルデータを集約しています。

## ディレクトリ構成

| パス | 内容 |
|------|------|
| [`spec/`](./spec/) | 画面・テーブル設計などの仕様 Markdown |
| [`hearing/`](./hearing/) | 要件定義 MTG などのヒアリング記録（テキスト等） |
| [`assets/images/`](./assets/images/) | UI 参考画像・図版（PNG 等） |
| [`reference/`](./reference/) | PDF・Word など参照ドキュメント |
| [`data/`](./data/) | Excel・CSV などモック・分析用データ |

## ファイル一覧（主要）

### spec

- `テーブルページ設計.md` — 一覧テーブル画面の設計
- `遷移先ページ設計.md` — 依頼者詳細（遷移先）の設計

モックアプリの総合仕様書（例: `モックアプリ開発仕様_グローウェル.md`）は **`spec/` に置く**想定です。バックアップから復元する場合も同パスを推奨します。
