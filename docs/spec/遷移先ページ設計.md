# 依頼者詳細ページ（遷移先）設計

## ページ概要

依頼者一覧テーブルの左端アイコン（🔍）をクリックすると遷移する、1依頼者ごとの詳細ページ。
5つのタブで情報をグループ分けして表示する。

---

## ページ全体レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│  ヘッダーバー（アプリ名 / ユーザー情報）                          │
├──────────┬───────────────────────────────────────────────────┤
│          │  ← 一覧に戻る    依頼者詳細：山田 太郎（ID: 001）  │
│  左      ├───────────────────────────────────────────────────┤
│  サイド  │  【タブナビゲーション】                              │
│  バー    │  [依頼者基本情報] [アポ・後確・面談] [和解処理]      │
│          │  [入金予定履歴]   [管理情報]                         │
│          ├───────────────────────────────────────────────────┤
│          │                                                    │
│          │  ← 選択中タブのコンテンツ →                        │
│          │                                                    │
└──────────┴───────────────────────────────────────────────────┘
```

### タブ共通のデザイン方針
- アクティブなタブは下線 + ブランドカラーで強調
- 各タブ内は**カード（白背景 + 薄い影）**でセクションを区切る
- セクション見出しは薄グレー帯 + 太字ラベル
- フィールドは「ラベル（小・グレー） ＋ 値（通常サイズ）」の縦並び
- テーブルはゼブラストライプ、ステータスはカラーバッジ表示

#### 和解ファイル（ファイルアップロードエリア）

マイナンバーカードの写真・委任状・和解書などの書類を依頼者ごとに一元管理するエリア。

**アップロードUI**

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│         📂  ファイルをここにドラッグ＆ドロップ          │
│              または  [ファイルを選択]                  │
│                                                      │
│    対応形式：JPG・PNG・PDF　最大ファイルサイズ：20MB   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**アップロード済みファイル一覧**

| 項目 | 仕様 |
|------|------|
| ファイル名 | アップロード時のファイル名をそのまま表示 |
| ファイル種別アイコン | 🖼️ 画像（JPG/PNG）/ 📄 PDF で視覚的に区別 |
| アップロード日時 | YYYY/MM/DD HH:MM |
| プレビューボタン（🔍） | クリックでモーダル内にプレビュー表示（画像はそのまま、PDFはビューア表示） |
| 削除ボタン（🗑） | クリックで確認ダイアログ表示後に削除 |

**対応ファイル形式と用途例**

| ファイル形式 | 用途例 |
|------------|--------|
| JPG / PNG | マイナンバーカード（表・裏）の写真、本人確認書類の写真 |
| PDF | 委任状、和解書、債権調査票、受任通知書など各種書類 |

**表示仕様**

- アップロード済みファイルはアップロード日時の降順（新しい順）で一覧表示
- ファイル名が長い場合は省略表示し、ホバーでフルネームを表示
- 複数ファイルの同時アップロード対応
- アップロード中はプログレスバーを表示
- アップロード完了後、一覧に即時反映

- 依頼者名（大きく表示）＋ IDを横並びで表示

### 避けるべきデザイン（再掲）

- 全フィールドを1画面に詰め込む（タブで分割すること）
- ラベルと値が区別しにくい均一なテキスト表示
- ステータスがテキストのみで色分けされていない
- テーブル内の情報が密集しすぎて読めない
