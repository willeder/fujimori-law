# 依頼者一覧テーブル

## 概要

- 各行の左端アイコン（🔍）をクリックすると、該当依頼者の詳細ページ（遷移先）へ移動する
- 各列はソート・フィルタ・検索に対応することを想定
- 横スクロール対応（列数が多いため）

---

## デザイン方針

> 参考画像：「希望しているUIのイメージ画像」2枚（CableBase EE / 住宅マネージャー）をもとに策定。
> 「好ましくないUI」2枚（旧来のWindows業務アプリスタイル）は**避けるべき見本**として参照。

### カラーパレット方針

| 用途 | 方向性 | 避けること |
|------|--------|-----------|
| ヘッダー・ナビ背景 | ネイビー〜ダークブルー系 | 銀色・グレー一色（旧Windowsスタイル） |
| アクセントカラー | ティール / ブルー系 | オレンジ・派手な原色の多用 |
| コンテンツ背景 | 白 / 薄いグレー | 濃いグレーの全面使い |
| ステータスバッジ | 状態に応じた色（下記参照） | テキストのみで色なし |
| テーブル行 | 白と薄グレーの交互（ゼブラ） | 罫線だらけの表 |

#### ステータス別カラーコーディング

| ステータス | バッジカラー | 用途例 |
|-----------|-------------|--------|
| 入金中 / 進行中 | 黄色系（amber） | 受任後ステータス、入金情報 |
| 完了 / 和解済 / 完済 | 緑系（green） | 受任後ステータス、入金情報 |
| 要注意 / 交渉中 | 青系（blue） | 受任後ステータス |
| 未入金 / 警告 | 赤系（red） | 入金情報、要注意ランク |
| なし / 通常 | グレー系 | デフォルト状態 |

### コンポーネント方針

#### テーブル
- 1行ごとに白 / 薄グレーを交互（ゼブラストライプ）
- ヘッダー行は背景色を濃くして視認性を確保
- 各セルは適切な余白（padding）を確保し、詰め込まない
- 列幅は内容に応じて柔軟に（長いテキスト列は省略 + ホバーでフル表示）
- ステータス系カラムは**カラーバッジ（丸みのあるラベル）**で表示

#### アイコンボタン（詳細リンク）
- 行の左端に配置、クリック可能であることが一目でわかるデザイン
- アイコン例：🔍 / 📄 / →（矢印）など
- ホバー時に色が変わるインタラクションを付与

#### ツールバー（テーブル上部）
- 検索ボックス：左寄せ、フリーワード入力
- フィルタ：ステータス・担当者・日付範囲などのドロップダウン
- 「ファイルに書き出す」ボタン：右寄せ、アイコン付き
- ボタン類は必要最低限に絞る（ツールバーを機能で埋め尽くさない）

#### セクション・カード
- 情報のグループはカード形式（白背景 + 薄いボーダー or 影）で区切る
- セクション見出しは背景色付きの帯（薄グレーまたはブランドカラー）で明示
- フォームのラベルはフィールドの上に小さく表示（フィールド内ラベルは避ける）

### タイポグラフィ方針

| 要素 | スタイル |
|------|---------|
| アプリ名・ページタイトル | 大きめ・太字・白抜き（ヘッダー内） |
| セクション見出し | 中サイズ・太字・ブランドカラーまたはダークグレー |
| テーブルヘッダー | 小〜中サイズ・太字・背景色あり |
| テーブルデータ | 小〜中サイズ・通常ウェイト・読みやすいサイズ |
| ラベル・注釈 | 小サイズ・グレー |
| フォント全般 | 日本語対応のシステムフォントまたはNoto Sans JP系 |

### 避けるべきデザイン（NG例）

以下は「好ましくないUIのイメージ画像」から抽出した**NGパターン**。

| NGパターン | 理由 |
|-----------|------|
| ウィンドウの重なり・多段モーダル | 現在位置が分からなくなる |
| 罫線だらけの密集したフォーム | 視認性が下がり、操作ミスを誘発 |
| グレー一色のWindows風ボタン・背景 | 古い印象、操作しにくい |
| ツールバーに大量のボタンを並べる | 何が主要な操作か分からなくなる |
| 全フィールドを1画面に詰め込む | 情報が多すぎて認知負荷が上がる |
| フィールドごとに太い枠線 | 見た目がうるさくなる |

## 動作仕様メモ

### 詳細リンクアイコン（左端）
- アイコン種別：🔍 または 📄 など視認性の高いアイコン
- クリック時の遷移先：各依頼者の詳細ページ（個別レコード画面）
- 遷移先ページには「依頼者基本情報」「和解対象債権一覧」「入金管理」等のタブ・セクションが含まれる

### 検索・フィルタ
- 各カラムでの絞り込み検索に対応
- フリーワード検索（名前・フリガナ・ID 等）
- ステータス別フィルタ（受任後ステータス / 入金情報 等）
- 日付範囲指定（受任日 / リスト登録日）

### ソート
- 全カラムで昇順・降順ソート対応
- デフォルトは受任日の降順（新しい順）

### ページネーション
- 1ページあたりの表示件数：20件（変更可能）

## 入金管理CSVカラム定義（「ファイルに書き出す」出力内容）

「ファイルに書き出す」ボタンを押すと、以下の構造のCSVが出力される。
1つの依頼者（ID）に対して**回数（支払い回数）分の行**が展開される形式。

| # | カラム名 | 型・形式 | 内容 |
|---|---------|---------|------|
| 1 | レコードの開始行 | フラグ（`*` / 空） | 各IDの先頭行に `*` が入る |
| 2 | ID | テキスト | 依頼者ID（例：102951E） |
| 3 | --（回数） | 数値 | 支払い回数の通し番号（1〜N回） |
| 4 | 累積[check] | 数値 | 累積チェックフラグ |
| 5 | 入金予定日 | 日付（YYYY/MM/DD） | 各回の入金予定日 |
| 6 | 入金予定額 | 金額（円） | 各回の入金予定総額 |
| 7 | 報酬充当予定額 | 金額（円） | うち事務所報酬へ充当予定の金額 |
| 8 | 弁代報酬充当予定額 | 金額（円） | うち弁護士費用へ充当予定の金額 |
| 9 | ﾌﾟｰﾙ充当予定額 | 金額（円） | うちプール（保留分）へ充当予定の金額 |
| 10 | 社数 | 数値 | 対象債権者数 |
| 11 | 手数料 | 金額（円） | 振込手数料等 |
| 12 | 弁済充当予定額 | 金額（円） | うち債権者への弁済に充当予定の金額 |
| 13 | 相違 | 金額（円） | 予定と実績の差異 |
| 14 | 実入金日 | 日付（YYYY/MM/DD） | 実際に入金された日付（未入金時は空） |
| 15 | 実入金額 | 金額（円） | 実際に入金された金額（未入金時は空） |
| 16 | 報酬充当額 | 金額（円） | 実績ベースの報酬充当額 |
| 17 | 弁代報酬充当額 | 金額（円） | 実績ベースの弁護士費用充当額 |
| 18 | ﾌﾟｰﾙ充当額 | 金額（円） | 実績ベースのプール充当額 |
| 19 | 弁済日 | 日付（YYYY/MM/DD） | 債権者への弁済実施日 |
| 20 | 数 | 数値 | 振込件数 |
| 21 | 振)手数料 | 金額（円） | 振込時に発生した手数料 |
| 22 | 弁済充当額 | 金額（円） | 実績ベースの弁済充当額 |
| 23 | 確認 | テキスト | 確認メモ |
| 24 | 額 | 金額（円） | 端数・調整額等 |
| 25 | check[check] | フラグ | チェック済フラグ |
| 26 | 入金check | フラグ / 数値 | 入金確認フラグ |
| 27 | 累積ﾌﾟｰﾙ | 金額（円） | 累積プール金額 |
| 28 | 実ﾌﾟｰﾙ | 金額（円） | 実プール金額（実績） |
| 29 | 受任日 | 日付（YYYY/MM/DD） | 案件の受任日 |
| 30 | 受任後ステータス | テキスト | 例：全社受任通知発送済 / 一部受任通知発送済 等 |

### CSVデータの構造メモ

- **1依頼者 = 複数行**：支払い回数ぶんの行が展開される（例：36回払いなら36行）
- **先頭行の識別**：`レコードの開始行` 列が `*` の行が各依頼者の1回目
- **実入金欄**：入金前は空欄。入金後に `実入金日` / `実入金額` 等が埋まる
- **ステータス例**：`全社受任通知発送済` / `一部受任通知発送済`
